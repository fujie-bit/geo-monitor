# 利欧 GEO 监测 — 部署与使用说明

这是一套**纯前端**的 GEO 可见度监测系统:你在后台手工录入数据 → 点「发布」导出一个数据文件 → 上传到托管覆盖旧文件 → 客户刷新即看到最新。零服务器、零账号、零月费。

---

## 一、文件说明

```
deploy/
├── index.html        ← 网站入口(必须叫这个名字)
├── app/              ← 程序文件夹(整个一起传,别拆)
└── geo-data.json     ← 发布的数据文件(首次没有也能跑,见下)
```

> ⚠️ 上传时要把 **index.html 和 app/ 文件夹一起传**,保持这个目录结构。

---

## 二、改密码(部署前必做)

1. 用记事本 / VS Code 打开 `app/app.jsx`
2. 顶部找到这几行,改成你自己的:
   ```js
   const CONFIG = {
     password: "geo2026",   // ← 改成你的后台密码
     agency: "利欧 GEO",     // ← 机构名(显示在页面上)
   };
   ```
3. 保存即可。

> 说明:这是「轻量密码门」,能挡住普通人随手打开,但**不是高强度安全**(懂技术的人能在源码里看到密码)。客户专属链接 `?c=xxx` 是公开只读的,不需要密码——这样客户点开就能看。如果以后要更强的权限控制,可以升级到带后端的版本。

---

## 三、免费托管 —— 推荐：GitHub + Netlify 自动发布

绑定 GitHub 后，更新数据**不用再拖文件夹**：把新的 `geo-data.json` 传到 GitHub，Netlify 会自动重新发布。全程网页点点点，**不需要装 Git、不用敲命令**。

### 一次性设置（约 10 分钟，只做一次）

**第 1 步 · 注册 GitHub**
打开 https://github.com → 注册免费账号。

**第 2 步 · 新建仓库**
1. 右上角 ➕ → New repository
2. 名字随便取，如 `geo-monitor`
3. 选 **Public**（公开；客户链接本就是公开只读）
4. Create repository

**第 3 步 · 上传部署文件**
1. 新仓库页面点 “uploading an existing file”（或 Add file → Upload files）
2. 把 `deploy/` **里面**的东西全选拖进去：`index.html`、`geo-data.json`、以及 `app` 整个文件夹
3. 底部点绿色 **Commit changes**
> ⚠️ 拖的是 deploy **里面的**文件，不要把 deploy 这层文件夹本身传上去；保持 `app/` 是个文件夹。

**第 4 步 · 把 Netlify 连到仓库**
1. 打开 https://app.netlify.com → 用 GitHub 账号登录
2. Add new site → Import an existing project → GitHub
3. 授权后选中刚建的 `geo-monitor` 仓库
4. 构建设置全部留空（纯静态站，不用填）→ Deploy
5. 几十秒后得到固定网址，如 `https://geo-monitor.netlify.app`

✅ 完成。以后**这个网址永远不变**。

### 以后每次更新数据（30 秒，无需命令）
```
①后台点「发布」→ 下载 geo-data.json
        ↓
②打开你的 GitHub 仓库页面
        ↓
③Add file → Upload files → 拖入新的 geo-data.json（同名覆盖）
        ↓
④点 Commit changes
        ↓
⑤Netlify 自动检测到变化，1 分钟内重新发布
        ↓
⑥客户刷新链接 → 看到最新数据 ✅
```
> 最省事：在仓库里**直接拖**新的 `geo-data.json` 进去，GitHub 看到同名文件会自动当成“更新”，Commit 一下即可。

### 备选方式 A — Netlify Drop（不绑 GitHub、纯拖拽）
1. 打开 https://app.netlify.com/drop
2. 把整个 **deploy 文件夹** 拖进网页 → 得到网址
3. 更新数据时：换掉文件夹里的 `geo-data.json`，把整个文件夹再拖一次

### 备选方式 B — GitHub Pages（不想用 Netlify 时）
1. 仓库 Settings → Pages → 选 main 分支 → 保存
2. 得到 `https://你的用户名.github.io/仓库名/` 网址
3. 更新数据：同样把新的 geo-data.json 传到仓库覆盖即可

## 四、日常使用流程

```
①打开网址 → 输密码进后台
        ↓
②「数据录入」录入/修改监测记录(实时算出指标)
   「客户管理」新增 / 编辑 / 删除客户
        ↓
③点右上角「↑ 发布数据」→ 浏览器下载 geo-data.json
        ↓
④把这个 geo-data.json 传到托管,覆盖旧的同名文件
        ↓
⑤客户刷新专属链接 → 看到最新数据 ✅
```

### 关于数据保存
- 你在后台的录入会**自动存在本机浏览器**里(换电脑 / 清缓存会丢)。
- **真正对外的数据 = 你上传到托管的 `geo-data.json`**。所以改完一定要「发布 + 上传」才会让客户看到。
- 换电脑工作时:用后台的「导入数据」按钮,选中当前的 `geo-data.json`,即可在新设备接着编辑。

---

## 五、给客户发链接

- 在「多客户总览」或「客户管理」里,点客户卡片的 **「复制专属链接」**。
- 链接形如:`https://你的网址/?c=durex`
- 客户打开**只能看到自己的数据**,只读、无需密码、手机电脑自适应。

---

## 六、新增一个客户

后台 →「数据录入后台」→「客户管理」→「+ 新增客户」,可配置:
- 客户名称 / 英文名 / 行业 / Logo 首字 / 主题色
- 竞品名单(用顿号或逗号分隔)
- 覆盖哪些 AI 平台(豆包 / DeepSeek / 元宝 / 千问 / Kimi / 文心,可勾选)
- 该客户的 Prompt 列表(可任意增删)

建好后到「数据录入」选中该客户,逐条录入监测结果。**完成后记得「发布」并上传。**

---

## 七、首次部署没有 geo-data.json 也能跑

系统内置了一份示例数据。首次部署即使没传 `geo-data.json`,客户链接也能看到示例。等你第一次「发布 + 上传」后,就会显示你的真实数据。

---

## 八、常见问题

**Q：客户说打开是空白 / 报错?**
确认 index.html 和 app/ 文件夹是一起上传的、目录结构没乱。

**Q：我改了数据,客户没看到?**
检查是否「发布」并把新的 geo-data.json 覆盖上传了;让客户强制刷新(Ctrl/Cmd + Shift + R)。

**Q：数据量会不会太大?**
几个客户 × 几十条 prompt 完全没问题。这套设计适合人工统计的轻量数据。
